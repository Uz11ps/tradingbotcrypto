## 1) Добавить понятную рекомендацию в каждый сигнал

Нужно добавить в каждый сигнал отдельное поле рекомендации:

- `Лонг`
- `Шорт`
- `Наблюдать`

### Логика рекомендации

Это не отдельный прогноз рынка и не новая аналитическая модель.
Это понятная интерпретация уже существующего сигнала.

Используем текущую логику проекта:
- если сигнал направлен вверх (`pump` / `up`) и action == `entry` → рекомендация `Лонг`
- если сигнал направлен вниз (`dump` / `down`) и action == `entry` → рекомендация `Шорт`
- во всех остальных случаях → `Наблюдать`

То есть:
- бот уже определил направление движения
- бот уже определил силу/тип действия (`entry/watch/hold`)
- мы просто переводим это в понятную торговую рекомендацию для пользователя

### Что нужно сделать

Добавить поле `recommendation` в API-схемы:
- `SignalOut`
- `LiveSignalOut`
- при необходимости `FeedMoverOut`

Вернуть `recommendation` из:
- `GET /signals`
- `POST /signals`
- `GET /signals/live`
- `GET /feed/movers`

В Telegram-сообщениях тоже показывать:
- `Рекомендация: Лонг`
- `Рекомендация: Шорт`
- `Рекомендация: Наблюдать`

### Важно
Сделай helper, чтобы логика не дублировалась по файлам.

Например создать файл:
- `app/services/signal_presentation.py`

И функции:
- `build_recommendation(direction: str | None = None, signal_type: str | None = None, action: str | None = None) -> str`
- `matches_signal_side_mode(mode: str, direction: str | None = None, signal_type: str | None = None) -> bool`

## 2) Упростить пользовательскую настройку минимального движения

Сейчас в UI триггер цены сделан неудобно — через два числа (5m и 15m).
Это нужно упростить.

### Новая логика
Оставить одну простую настройку:

- `min_price_move_pct`
- в UI показывать как:
  - `Минимальное движение: ≥ 5.0%`

### Смысл
Это не жёстко зашитый фильтр на 10%.
Это пользовательская настройка, которую можно менять вручную:
- 5%
- 10%
- 15%
- и т.д.

Она нужна, чтобы быстро отсортировать рынок под себя:
- хочешь видеть больше сигналов — ставишь 5
- хочешь только сильные движения — ставишь 10 или 15

### Что нужно сделать в UI
- убрать пользовательский ввод формата `5m 15m`
- заменить на ввод одного числа
- пример:
  - `5`
  - `10`
  - `15`

В экране настроек показывать:
- `Минимальное движение: ≥ X%`

В reset настроек дефолт должен быть:
- `5.0`

### Что нужно сделать в backend
Использовать `min_price_move_pct` как единый пользовательский фильтр.

Логика:
- если абсолютное движение сигнала меньше `min_price_move_pct`, сигнал не должен попадать пользователю
- если больше или равно — показываем

### Где это должно реально работать
Фильтрация по `min_price_move_pct` должна работать в:
- RSI-воркере перед сохранением и отправкой сигнала
- `/feed/movers` если передан `chat_id`
- при необходимости в legacy feed тоже
- если есть список сигналов для пользователя — там тоже учитывать

### Важно
Не ломай текущую БД.
Поле `min_price_move_pct` уже есть — используй его как основную простую настройку.

Если внутри системы всё ещё живут старые 5m/15m threshold’ы, можешь оставить их как внутреннюю техническую часть, но итоговый пользовательский фильтр должен работать именно по одному понятному числу:
- `min_price_move_pct`

То есть пользователь управляет именно этой одной настройкой.

## 3) Разграничить режимы: всё / только pump / только dump

Нужна новая пользовательская настройка режима направления сигналов.

### Значения
- `all`
- `pump`
- `dump`

### Смысл
- `all` — показывать и pump, и dump
- `pump` — показывать только растущие сигналы
- `dump` — показывать только падающие сигналы

### Что нужно сделать
Добавить поле в `UserSignalSettings`:
- `signal_side_mode: str | None`

Допустимые значения:
- `all`
- `pump`
- `dump`

### Изменить

- `app/db/init.py`
  - через additive schema update:
    - `ALTER TABLE user_signal_settings ADD COLUMN IF NOT EXISTS signal_side_mode VARCHAR(16)`

Обновить:
- `EffectiveUserSettings`
- `get_global_defaults()`
- `get_effective_settings()`
- `upsert_user_settings()`

Обновить API-схемы:
- `UserSignalSettingsUpdate`
- `UserSignalSettingsOut`

Обновить:
- `app/bot/api_client.py`

### В Telegram UI
Добавить кнопку:
- `Направление`

По нажатию показывать inline keyboard:
- `Все`
- `Только pump`
- `Только dump`

Показывать текущее значение в настройках.

### Где фильтрация должна работать
- в RSI-воркере перед сохранением/отправкой сигнала
- в `/feed/movers`, если передан `chat_id`
- в legacy feed при необходимости

### Логика фильтрации
- `all` → пропускаем всё
- `pump` → только сигналы с `signal_type == "pump"` или `direction == "up"`
- `dump` → только сигналы с `signal_type == "dump"` или `direction == "down"`
